# Запуск Telegram-бота на BotHost (bothost.ru)

Бот на **aiogram 3**, работает по **long polling** — домен, webhook и белый IP не нужны.

## 1. Что загрузить
Загрузите всю папку `bot/`:
```
bot/
├─ bot.py            ← главный файл запуска
├─ config.py
├─ db.py
├─ exporter.py
└─ requirements.txt  ← зависимости (ставятся автоматически)
```

## 2. Настройки проекта
1. Новый проект → тип **Python-бот**.
2. **Версия Python:** 3.11 (или 3.10+).
3. **Команда запуска / главный файл:** `python bot.py`
4. **Зависимости:** bothost сам выполнит `pip install -r requirements.txt`. Если нужно вручную — в консоли: `pip install -r requirements.txt`.

`requirements.txt`:
```
aiogram==3.13.1
PyMySQL==1.1.1
openpyxl==3.1.5
python-dotenv==1.0.1
aiohttp_socks==0.10.1
```

## 3. Переменные окружения (Variables / Env)
Задайте в разделе переменных проекта (на bothost файл `.env` не обязателен):

| Переменная | Обяз. | Описание |
|---|---|---|
| `TG_BOT_TOKEN` | да | Токен от @BotFather |
| `TG_ADMIN_IDS` | да | ID админов через запятую, напр. `111,222`. Им идут заявки и кнопки одобрения |
| `DB_HOST` | да | Хост MySQL сайта |
| `DB_PORT` | нет | Порт (по умолчанию 3306) |
| `DB_DATABASE` | да | Имя БД |
| `DB_USERNAME` | да | Пользователь БД |
| `DB_PASSWORD` | да | Пароль БД |
| `TG_PROXY` | нет | SOCKS5-прокси `socks5://user:pass@host:port`, если Telegram заблокирован |
| `EXPORT_CHAT_ID` | нет | Чат для авто-выгрузки Excel |
| `EXPORT_INTERVAL_MIN` | нет | Период выгрузки (мин, по умолчанию 60) |

> **Важно:** бот и сайт должны смотреть в **одну базу данных** — через неё передаются уведомления (таблица `bot_outbox`) и заявки.

## 4. Запуск
Нажмите **Start**. В логах появится `Подключился как @ваш_бот`. Бот сам переподключается при обрывах сети.

## 5. Что бот делает (уведомления с сайта)
Бот раз в ~6 сек читает очередь `bot_outbox`:

**Сотруднику** (в личку, если привязал Telegram): ⬆️ повышение / ⬇️ понижение, ✅ «актив учтён», 🏅 добавлены/убраны знаки, 📝 выдан тест, результат по его заявке/объяснительной.

**Админам** (из `TG_ADMIN_IDS`): 📝 новые заявки, 📑 объяснительные, 📄 отчёты — с кнопками **✅ Одобрить / ❌ Отклонить**, а также вся модерация.

**Одобрение в боте:** кнопка сразу меняет статус на сайте, уведомляет сотрудника и удаляет сообщение с кнопками у всех админов. Если заявку уже рассмотрели **на сайте**, сообщение с кнопками в боте автоматически удаляется.

## 6. Привязка аккаунта
Сотрудник получает личные уведомления только после привязки: на сайте в профиле взять код, в боте отправить `/start <код>`.
